## 1️⃣ Examples for a **Query Parameter** (`categoryId`)

```yaml
parameters:
  - name: categoryId
    in: query
    required: false
    schema:
      type: integer
    examples:
      mobiles:
        summary: Use this example for mobiles category
        value: 101
      laptops:
        summary: Use this example for laptops category
        value: 102
      headphones:
        summary: Use this example for headphones category
        value: 103
      televisions:
        summary: Use this example for televisions category
        value: 104
```

This creates a **dropdown of example values** in Swagger UI for the same parameter.

## 2️⃣ Examples for a **Response Returning All Categories**

```yaml
responses:
  '200':
    description: Successful response
    content:
      application/json:
        schema:
          type: array
          items:
            type: object
            properties:
              categoryId:
                type: integer
              name:
                type: string
        examples:
          all-categories:
            summary: Response when no categoryId is provided
            value:
              - categoryId: 101
                name: Mobiles
              - categoryId: 102
                name: Laptops
              - categoryId: 103
                name: Headphones
              - categoryId: 104
                name: Televisions
          single-category:
            summary: Response when categoryId is provided
            value:
              - categoryId: 101
                name: Mobiles
```

## 3️⃣ Examples for a **Products API Response**

```yaml
responses:
  '200':
    description: Successful response
    content:
      application/json:
        schema:
          type: array
          items:
            type: object
            properties:
              productName:
                type: string
              price:
                type: number
              category:
                type: string
              quantity:
                type: integer
        examples:
          products-data:
            summary: Sample products response
            value:
              - productName: iPhone 15
                price: 1200
                category: Mobiles
                quantity: 10
              - productName: Samsung TV
                price: 900
                category: Televisions
                quantity: 5
```

---

## Key Rules Applied (important for correctness)

* `examples` is always placed **at the same level as `schema`**
* Each example has:

    * a **name** (free text, for UI only)
    * an optional `summary`
    * a `value` that **must match the schema**
* Works in:

    * parameters
    * request bodies
    * responses
