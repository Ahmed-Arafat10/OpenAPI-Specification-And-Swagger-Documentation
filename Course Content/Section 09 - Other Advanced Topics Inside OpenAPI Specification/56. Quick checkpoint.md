## Operations (HTTP Method Objects)

Each HTTP method inside a path is referred to as an **Operation**.

Supported HTTP methods include:

* `get`
* `put`
* `post`
* `delete`
* `patch`
* `head`
* `options`
* `trace`

Each of these represents a **distinct operation** within a path.

## Servers Object and Priority Rules

### Where Servers Can Be Defined

Server information can be defined at **multiple levels**:

1. **Root level**
2. **Path level**
3. **Operation (HTTP method) level**

### Server Priority Behavior

When servers are defined in multiple places:

* **Operation-level server definitions take highest priority**
* They override:

    * Path-level servers
    * Root-level servers

This allows:

* Global defaults at the root
* Path-specific overrides
* Operation-specific routing when needed

* The `deprecated` field is available at the **operation level**
* It marks an API operation as **no longer recommended for use**

## Callbacks Object

* Used for **asynchronous API workflows**
* Applicable when:

    * An API processes a request
    * Responds later via a callback endpoint
* If your API is **synchronous**, this object can be safely ignored
* It is an **optional object**
