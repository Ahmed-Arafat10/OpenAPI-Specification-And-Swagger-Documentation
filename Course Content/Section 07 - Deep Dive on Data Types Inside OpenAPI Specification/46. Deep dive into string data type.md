## 46. Deep Dive into `string` Data Type

In this lecture, the focus is on the **most flexible and powerful data type in OpenAPI**: `string`.

Although `string` looks simple at first glance, it can represent **many real-world data types** such as dates, timestamps, passwords, and even binary/file data—**all with the help of the `format` keyword**.

---

## Basic `string` Usage

At the simplest level, `string` represents textual data.

### YAML Example — Simple String

```yaml
schema:
  type: string
```

Used for:

* names
* titles
* descriptions
* identifiers
* free-form text

---

## Extending `string` Using `format`

The **real power** of the `string` type comes from the `format` keyword.

Using `format`, a `string` can behave like:

* date
* date-time
* password
* binary/file
* email
* IP address
* hostname
* and more (tool-dependent)

---

## Date Representation

### `format: date`

Represents a **full date only** (no time).

### YAML Example — Date

```yaml
schema:
  type: string
  format: date
```

Example value:

```
2024-11-20
```

Swagger UI and tools will treat this as a **date**, not a generic string.

---

## Date & Time Representation

### `format: date-time`

Represents **date + time** (RFC 3339 format).

### YAML Example — Date-Time

```yaml
schema:
  type: string
  format: date-time
```

Example value:

```
2024-11-20T14:35:10Z
```

Useful for:

* timestamps
* createdAt / updatedAt fields
* logs
* audit trails

---

## Password Representation

### `format: password`

Indicates **sensitive input**.

Swagger UI:

* masks input values
* shows dots instead of characters

### YAML Example — Password

```yaml
schema:
  type: string
  format: password
```

Used for:

* login APIs
* credential submission
* secret inputs

---

## File / Binary Data Representation

When dealing with:

* files
* images
* binary payloads
* base64-encoded data

You can use:

### `format: byte` (Base64 encoded)

```yaml
schema:
  type: string
  format: byte
```

### `format: binary` (Raw binary data)

```yaml
schema:
  type: string
  format: binary
```

These are commonly used in:

* file upload APIs
* document/image services

---

## Tool-Supported (Non-Official but Widely Used) Formats

These formats are **not strictly enforced by the OpenAPI spec**, but many tools support them:

| Format     | Meaning                       |
| ---------- | ----------------------------- |
| `email`    | Email address                 |
| `uuid`     | Universally unique identifier |
| `hostname` | Hostname                      |
| `ipv4`     | IPv4 address                  |
| `ipv6`     | IPv6 address                  |
| `uri`      | URL / URI                     |

> ⚠️ If a tool does not support these formats, it will **silently fall back to string**.

### YAML Example — Email

```yaml
schema:
  type: string
  format: email
```

---

## String Length Constraints

You can restrict string length using:

* `minLength`
* `maxLength`

### YAML Example — Length Constraints

```yaml
schema:
  type: string
  minLength: 3
  maxLength: 50
```

Used for:

* usernames
* titles
* codes
* identifiers

---

## Pattern Validation (Regular Expressions)

The `pattern` keyword allows you to enforce **custom formats using regex**.

### Example — SSN Pattern

```yaml
schema:
  type: string
  pattern: '^\d{3}-\d{2}-\d{4}$'
```

Valid values:

```
123-45-6789
```

Invalid values:

```
123456789
12-345-6789
```

---

### Example — Alphanumeric Only

```yaml
schema:
  type: string
  pattern: '^[a-zA-Z0-9]+$'
```

---

## Practical Example — Product Release Date

### Product Schema with Date

```yaml
Product:
  type: object
  properties:
    name:
      type: string
    releaseDate:
      type: string
      format: date
```

Swagger UI behavior:

* Displays date picker
* Auto-populates today’s date if example is missing

---

## Switching to Date-Time (If Needed)

```yaml
releaseDate:
  type: string
  format: date-time
```

Swagger UI now shows:

* date + time
* timezone-aware timestamp

---

## Important Notes from This Lecture

* `string` is **not just text**
* `format` defines **semantic meaning**
* Tools rely heavily on `format`
* Unsupported formats fall back safely to `string`
* Use `minLength`, `maxLength`, and `pattern` for validation
* Date, password, and binary types are all achieved via `string`

> **The `string` data type is the most flexible data type in OpenAPI.
