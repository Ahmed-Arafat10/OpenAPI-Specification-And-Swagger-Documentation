# 7. History of OpenAPI & Swagger

## 1. Purpose of This Lecture

This lecture clears up one of the **most common confusions in the developer community**:

> What is Swagger?
> What is OpenAPI Specification?
> Are they the same thing?

To answer this correctly, we need to understand the **history and evolution** of both.

---

## 2. The Industry Problem Before 2010

Before 2010:

* There was **no standard** way to describe APIs
* Every organization documented APIs differently
* Tooling and automation were nearly impossible
* Developers struggled with unclear API contracts

A small group of developers recognized that the industry **urgently needed a standard** for REST API definitions.

---

## 3. 2010: Birth of Swagger (Open Source Initiative)

Around **2010**, a **small open-source community** started working on:

* A **specification** for defining RESTful APIs
* A **set of tools** to work with that specification

These tools included:

* Swagger Editor
* Swagger UI
* Swagger Codegen

The goal was to:

* Design APIs
* Describe APIs
* Visualize APIs
* Generate code from API definitions

At this time:

* The **specification**
* The **tools**
* The **name**

were all called **Swagger**.

So historically:

> **Swagger = specification + tools**

This is why many developers still use the term *Swagger* to mean the API standard.

---

## 4. Why the Confusion Still Exists

Because from **2010 to 2015**:

* The API specification itself was called **Swagger Specification**
* Everyone referred to API definitions simply as “Swagger”

This habit carried forward, and many developers still use:

* “Swagger”
* “OpenAPI”

interchangeably — even though they are **not the same thing today**.

---

## 5. 2015: SmartBear Acquires Swagger

In **2015**, **SmartBear** adopted Swagger.

What SmartBear did is very important:

### 5.1 Donation of the Specification

* SmartBear **donated the specification part** to the open-source community
* This was done under the umbrella of the **Linux Foundation**
* The specification was **renamed** from:

    * *Swagger Specification*
    * to **OpenAPI Specification**

Why the rename?

* SmartBear wanted to **retain trademark and copyright** over the name *Swagger*
* The specification itself became vendor-neutral

---

## 6. What Changed After 2015?

After 2015, there was a **clear separation**:

### 6.1 OpenAPI Specification (OAS)

* The **standard**
* Governed by the open-source community
* Evolved under the Linux Foundation
* Vendor-neutral
* Defines **how APIs should be described**

### 6.2 Swagger

* The **tooling ecosystem**
* Owned and maintained by SmartBear
* Includes:

    * Swagger Editor
    * Swagger UI
    * Swagger Codegen
* Tools are still **open source**

SmartBear continuously updates Swagger tools to stay **in sync with OpenAPI Specification changes**.

---

## 7. Current State (Then vs Now)

### Before 2015

* Swagger = Specification + Tools

### After 2015

* **OpenAPI Specification** = API standard
* **Swagger** = Tools built around that standard

This distinction is critical.

## 8. SmartBear’s Role Today

Beyond open-source tools, SmartBear also provides:

* Commercial products
* Enterprise-grade API tooling

These help large organizations:

* Manage OpenAPI at scale
* Govern APIs
* Improve collaboration and automation

This is one of the major advantages SmartBear gained by acquiring Swagger.

---

## 9. Correct Terminology (Very Important)

From a **technical and industry-correct perspective**:

* **OpenAPI Specification**

    * Refers to the **standard**
    * The contract / definition format

* **Swagger**

    * Refers to **tools**
    * Editor, UI, Codegen, and related utilities

### Real-World Reality

Many developers (including the instructor) still:

* Use the terms interchangeably
* Say “Swagger” when they mean OpenAPI

This is due to **historical habits**, not correctness.

---

## 10. Key Mental Model to Keep

* **OpenAPI Specification** → *What your API looks like*
* **Swagger tools** → *What helps you work with that definition*

If you remember this separation, the confusion disappears completely.

---

## 11. Additional Important Note

**(Added by me – not mentioned in the lecture)**

> In modern documentation and tooling ecosystems, many non-SmartBear tools (like validators, generators, and gateways) support **OpenAPI Specification directly**, proving that the standard has successfully outgrown its Swagger origins.
