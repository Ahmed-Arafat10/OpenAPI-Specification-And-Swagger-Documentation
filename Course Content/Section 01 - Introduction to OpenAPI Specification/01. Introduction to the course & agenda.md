## 2. What Is OpenAPI Specification?

**OpenAPI Specification (OAS)** is an **industry-standard format** used to **describe RESTful APIs** in a consistent, machine-readable way.

It allows you to define:

* API endpoints
* Request and response formats
* Parameters and validations
* Authentication & authorization
* Error responses
* API metadata

The specification is maintained by the **OpenAPI Initiative** and is widely adopted across the industry.

---

## 3. Why Should APIs Be Described Using OpenAPI?

In real-world environments:

* APIs are often consumed by **multiple clients**
* APIs may be shared across **teams or organizations**
* APIs evolve over time

This creates recurring questions from:

* API consumers
* Frontend teams
* QA teams
* External partners

Typical questions include:

* What does the request body look like?
* What is the response structure?
* What authentication is required?
* What validations are enforced?
* What error responses are possible?

### The Problem

You **cannot manually explain APIs** to everyone, every time.

### The Solution

Describe your APIs once using **OpenAPI Specification**, and let the documentation:

* Answer all questions
* Stay consistent
* Act as a single source of truth

---

## 4. Role of Swagger Tools

Once you define APIs using OpenAPI Specification, **Swagger tools** help you:

* Visualize API documentation
* Test APIs interactively
* Generate client code
* Generate server-side skeletons
* Validate API definitions

Swagger tools exist in:

* **Open-source** variants
* **Commercial** (enterprise-grade) offerings

Mastering both OpenAPI and Swagger tools **dramatically simplifies API development and collaboration**.

## 6. OpenAPI Development Approaches

You will learn OpenAPI usage in **two real-world scenarios**:

### 6.1 Code-First Approach

* Backend code already exists
* You want to **document existing APIs**
* OpenAPI spec is written after implementation

### 6.2 Design-First Approach

* API requirements come from a client
* APIs must be **designed and documented first**
* Backend development starts **after** OpenAPI definition

Both approaches are common in industry, and the course teaches how to handle **each correctly**.
